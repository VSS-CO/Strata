Welcome to Strata

Strata is a modern, type-safe scripting language that combines the best features from C, Go, Rust, and TypeScript. It compiles to C code or runs directly with a fast interpreter.

Why Strata?

Strata was created to solve a problem: most scripting languages sacrifice type safety for convenience, while systems languages require verbose boilerplate. Strata aims for the middle ground:

  • Type-Safe: Catch errors at compile time, not runtime
  • Clean Syntax: No unnecessary verbosity or magic
  • Immutable by Default: Safer code with less mutation
  • Compiles to C: Generate readable C code you can inspect
  • Simple: The entire compiler fits in one file with zero dependencies

Key Characteristics

Aspect Feature
Type System Static, explicit annotations required
Type Checking Compile-time (before execution)
Paradigm Procedural with functional elements
Mutability Immutable by default (let/const)
Memory Automatic (in interpreter), C-native (when compiled)
Modules File-based with explicit imports
Compilation To C or direct interpretation

A Quick Example

import io from std::io import math from std::math // Function with explicit types func fibonacci(n: int) => int { if (n <= 1) { return n } return fibonacci(n - 1) + fibonacci(n - 2) } // Type-safe immutable variables let result: int = fibonacci(10) io.print(result)

This code demonstrates Strata's core features: explicit type annotations, immutable variables by default, module imports, and functions with clear signatures.

Language Principles

Explicit Over Implicit

All imports are explicit, all types are explicit, all mutations use the var keyword. This makes code intentions clear and enables optimizations.

Type Safety First

Type checking happens at compile time before execution. Errors are caught early when they're cheap to fix.

Safe Defaults

Variables are immutable (let/const) by default. Mutation requires the explicit var keyword.

Simple Implementation

The entire compiler is ~600 lines in one file. Easy to understand, modify, and deploy.
