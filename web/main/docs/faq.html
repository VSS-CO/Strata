Frequently Asked Questions

What is Strata?

Strata is a modern, statically-typed scripting language that combines the best features from C, Go, Rust, and TypeScript. It compiles to C code or runs directly with a fast interpreter.

Is Strata production-ready?

Strata is stable and feature-complete for its v1.0 scope. The core language features are all working and tested. The single-file implementation makes it easy to understand and modify.

Can I use Strata for web development?

Not yet. Strata currently targets command-line applications and system tools. Web support is planned as a future extension.

How does Strata compare to Go or Rust?

Strata is simpler than both. It's meant for scripts and small programs, while Go and Rust are designed for larger systems. Strata has an easier learning curve and faster iteration.

Can I compile Strata to JavaScript?

Yes! Set target = "js" in your strata.toml to compile to JavaScript instead of C.

Is the compiler written in Strata?

No, the compiler is written in TypeScript (620+ lines, single file). This makes it easy to understand and modify.

How do I handle errors?

Error handling is still being designed. Currently Strata uses return values. Future versions will support Result types and error propagation.

Can I create my own modules?

Yes! Strata supports both relative imports and package imports. See the Modules documentation for details.

What about null/nil?

Strata doesn't have null values. All variables are initialized. Optional types are planned for future versions.

Where do I report bugs?

Report issues on GitHub. Pull requests are welcome!
