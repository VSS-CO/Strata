Variables

Strata has three keywords for binding values to names: let, const, and var. Each has different properties for immutability and scope.

let - Immutable Binding

Use let to create an immutable binding. Once assigned, the value cannot be changed:

let name: string = "Alice" let age: int = 30 let score: float = 95.5

Attempting to reassign a let binding is a compile-time error:

let x: int = 10 let x: int = 20 // ERROR: cannot reassign immutable binding

const - Compile-Time Constant

Use const for compile-time constants. These are optimized away and the value must be a literal:

const PI: float = 3.14159 const MAX_SIZE: int = 1000 const APP_NAME: string = "Strata"
const vs let: Use const for named values that are truly constant (magic numbers, configuration). Use let for regular immutable variables that may depend on computation.

var - Mutable Variable

Use var when you need to reassign a value. This is explicit about mutability:

var counter: int = 0 var counter: int = counter + 1 var counter: int = counter + 1 io.print(counter) // 2
Reassignment Requires Shadowing: To change a var, you must use var name: Type = new_value. This makes mutations explicit in your code.

Scope

Variables are scoped to the block they're declared in. Blocks are created by function bodies and control flow statements:

let global: int = 1 func example() => void { let local: int = 2 // Only visible in this function io.print(global) // OK: can access outer scope io.print(local) // OK: can access own scope } // io.print(local) // ERROR: local is not defined here

Block Scope with Control Flow

let x: int = 1 if (x > 0) { let y: int = 2 // Only visible inside if block io.print(y) } // io.print(y) // ERROR: y is not in scope

Immutability by Default

Strata prefers immutability. The default is let, not var. This encourages you to think about when mutation is really needed:

Binding Mutability Use Case
let Immutable Default choice for most variables
const Immutable Named constants and magic numbers
var Mutable Only when you need to reassign
Why Immutability by Default? Immutable variables are easier to reason about. They prevent accidental mutations and make code more predictable. Use var only when mutation is essential.

Type Annotations

All variable declarations require explicit type annotations. This makes your intent clear:

let age: int = 25 // Good: type is clear // let age = 25 // ERROR: type annotation required

Examples

Counter with var

var count: int = 0 var count: int = count + 1 var count: int = count + 1 var count: int = count + 1 io.print(count) // 3

Mixing let and var

let name: string = "Alice" // Immutable var balance: float = 100.0 // Mutable var balance: float = balance - 10.0 // Can change balance // var name: string = "Bob" // ERROR: can't change immutable let

Best Practices

  • Use let by default for all variables
  • Use const for named constants (MAX_SIZE, PI, etc.)
  • Use var only when you need to reassign
  • Always include explicit type annotations
  • Choose descriptive variable names
  • Keep variables scoped as narrowly as possible
